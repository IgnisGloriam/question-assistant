# Теоретическая часть курсовой

## Какие темы нужно покрыть

Твоя работа стоит на пересечении нескольких областей. Теоретическая глава обычно идёт от общего к частному:

```
1. Автоматическая генерация вопросов (Automatic Question Generation)
   └── Зачем это нужно, какие подходы существуют, история

2. Большие языковые модели (Large Language Models)
   └── Что такое трансформер, как работает генерация текста
   └── Квантизация моделей (почему можно запускать локально)

3. Retrieval-Augmented Generation (RAG)
   └── Зачем нужен, из чего состоит
   └── Эмбеддинги и векторный поиск

4. Методы повышения качества генерации
   └── Best-of-N sampling
   └── LLM-as-Judge
   └── Self-Refinement

5. Таксономия Блума
   └── Уровни когнитивных целей
   └── Связь с типами тестовых вопросов
```

## Сколько теории нужно

Зависит от требований к курсовой, но ориентир:

```
Теоретическая часть:    15-25 страниц
Практическая часть:     15-20 страниц
Введение + заключение:  3-5 страниц
─────────────────────────────────────
Итого:                  ~40-50 страниц
```

Не нужно пересказывать учебник по нейросетям на 50 страниц. Достаточно описать каждую технологию на 3-5 страниц: **что это → зачем нужно в твоей работе → как работает на концептуальном уровне**.

## Где искать

### Научные статьи (основа теории)

**Google Scholar** — главный поисковик по научным работам:

```
https://scholar.google.com
```

Ключевые запросы для поиска:

```
"automatic question generation" LLM
"retrieval augmented generation" survey
"LLM as judge"
"best-of-n sampling" language model
"test question generation" transformer
"automatic quiz generation"
"natural language generation" education
```

### Конкретные статьи, которые стоит прочитать

**По RAG:**

- Lewis et al. — *"Retrieval-Augmented Generation for Knowledge-Intensive NLP Tasks"* (2020) — оригинальная статья, придумавшая термин RAG

**По LLM-as-Judge:**

- Zheng et al. — *"Judging LLM-as-a-Judge"* (2023) — подробно про подход с моделью-судьёй

**По генерации вопросов:**

- Kurdi et al. — *"A Systematic Review of Automatic Question Generation for Educational Purposes"* (2020) — обзор всего поля
- Elkins et al. — *"How Useful Are Educational Questions Generated by Large Language Models?"* (2024) — свежее, прямо про LLM

**По архитектуре трансформера:**

- Vaswani et al. — *"Attention Is All You Need"* (2017) — оригинальная статья про трансформер

**По таксономии Блума:**

- Anderson & Krathwohl — *"A Taxonomy for Learning, Teaching, and Assessing"* (2001)

## Примерная структура теоретической главы

```
Глава 1. Теоретические основы

1.1. Автоматическая генерация тестовых вопросов        (3-4 стр.)
     - Проблема ручного составления тестов
     - Эволюция подходов: шаблоны → NLP → LLM
     - Критерии качества тестового вопроса

1.2. Большие языковые модели                           (5-6 стр.)
     - Архитектура трансформера (кратко)
     - Авторегрессионная генерация текста
     - Инструктивные модели (instruction-tuned)
     - Квантизация: зачем и как работает
     - Обзор модели Qwen2.5

1.3. Retrieval-Augmented Generation                    (4-5 стр.)
     - Проблема ограниченного контекста и галлюцинаций
     - Архитектура RAG: retrieval + generation
     - Векторные представления текста (эмбеддинги)
     - Векторные базы данных

1.4. Методы повышения качества генерации               (3-4 стр.)
     - Температура и сэмплирование
     - Best-of-N sampling
     - LLM-as-Judge
     - Self-Refinement

1.5. Таксономия Блума в контексте тестирования         (2-3 стр.)
     - Уровни когнитивных целей
     - Применение к автоматической генерации вопросов
```

## Практический совет

Пиши теорию **после** того как сделаешь практическую часть. Тогда ты уже будешь понимать, что реально используешь, и не напишешь лишнего. Каждый теоретический раздел должен отвечать на вопрос: **«Почему в моей системе используется именно это?»**
